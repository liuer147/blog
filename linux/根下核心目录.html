    根下核心目录

    # 根下核心目录

    • /bin => /usr/bin：binary二进制文件，命令
    • /sbin => /usr/sbin：super binary超集命令，只有root可以使用
    • /boot：与linux系统启动与引导相关的文件存放处：存放引导系统启动程序，系统内核镜像
    • /dev：device存放设备文件，光盘，硬盘分区，U盘 /dev/cdrom
    • /etc：系统的配置文件存放目录，很多自己安装的软件的配置文件也存放在这里
    • /home：/home/xxx普通用户的家目录
    • /root： root用户的家目录
    • /lib、/lib64：library库文件（服务软件的依赖）
    • /mnt：mount tempoary默认的一个挂载点，临时挂载点（磁盘），U盘、光盘的临时入口。目录默认为空，通过此目录下的某些文件连接设备
    • /opt：option第三方软件安装位置
    • /proc：process进程，虚拟目录，存放的是内存中信息：进程，进程信息，内核信息
    • /sys：和/proc类似，是虚拟目录，存放内存中的信息
    • /tmp：temporary临时文件存放的位置
    • /usr：user存放用户安装的软件
    • /var：variable存放经常变换的数据：/var/log日志(服务日志)

    # Linux核心目录的核心文件概述

    # /etc目录

    # /etc/hosts：主机ip地址与域名(主机名)的对应关系

    • /etc/hosts的作用：通过修改其文件，增加对应ip地址与域名的映射后，我们本机访问对应的域名时，就不再是原先的ip了，而是我们配置ip

    # /var目录：经常变化的内容，日志

    # /var/log/messages

    系统通用日志，各种信息默认都会写入到这个文件中

    # /var/log/secure

    用户安全日志，用户 登录记录 信息

    # /proc目录

    系统服务信息，进程信息，内核信息，系统信息...

    # /proc/cpuinfo：cpu信息

    • processor：cpu核心的id，从0开始
    • physical id：物理cpu的id号，从0开始

    使用lscpu命令也可以打印类似信息，我们需要关注：

    • CPU(s)：cpu的核心总数
    • Socket(s)：cpu颗数

    # /proc/meminfo：内存信息

    • MemTotal：总内存
    • MemFree：可用内存
