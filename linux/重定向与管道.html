    # 标准输出重定向

    将标准输出重定向到其他文件，而非出现在屏幕上，可以使用重定向操作符 >。

    将/usr/bin目录的文件列表输出到ls_output.txt文件
ls -l /usr/bin > ls_output.txt
    1
    2

    注意： 1、如果命令出错了，例如上述的ls -l /usr/bin换成了ls -l /aaa/bin，此时/aaa/bin目录并不存在，此时错误信息仍然会显示在屏幕上，而不是重定向到ls_output.txt文件！为什么？

    答：ls命令的错误并没有将错误消息发送到标准输出，而是将错误消息发送到了标准错误。因为我们只是重定向了标准输出，并没有重定向标准错误。

    2、当出现错误之后，我们再次执行命令ls -l ls_output.txt查看目标文件的信息。发现，文件大小为0！为什么？

    答：当使用重定向符>对标准输出进行重定向时，会 完全重写 目标文件。由于ls命令出错了，其除了错误消息外并没有产生任何输出结果。重定向操作在一开始准备 重写(清空文件内容) 该文件时，出现了错误终止了写入，所以该文件大小为0。

    # 清空、创建文件

    > ls_output.txt // 清空文件内容
> new_file.txt // 创建新文件

    2

    # 将标准输出追加到文件末尾

    可以使用 >> 来完成此功能。

    ls -l /usr/bin >> ls_output.txt
    1

    # 标准错误重定向

    标准错误重定向缺少专门的重定向操作符。我们需要借助文件描述符来实现。 描述符对应文件流：

    0 标准输入
    1 标准输出
    2 标准错误

    我们可以使用以下写法来重定向标准错误，文件描述符2紧挨着重定向操作符，将标准错误输出到ls_output.txt

    ls -l /bin/usr 2> ls_error.txt
    1

    # 将标准输出_和_标准错误重定向到同一个文件

    有两种方式：

    • ls -l /aaa/bin > ls_output.txt 2>&1：适用于旧版shell。我们执行了两次重定向：先重定向标准输出到ls_output.txt，然后使用2>&1将文件描述符2 (标准错误)重定向到文件描述符1 (标准输出)。

    注意： 重定向操作的顺序很重要！

    • > ls_output.txt 2>&1
    • 2>&1 > ls_output.txt：标准错误会被重定向到屏幕
    • ls -l /aaa/bin &> ls_output.txt：联合操作

    # 丢弃无用的输出结果

    系统提供了一种方法：将输出结果重定向到名为/dev/null的特殊文件，来实现丢弃无用输出。

    // 丢弃无用输出
ls -l /bin/usr 2> /dev/null

    # 标准输入重定向

    介绍标准输入重定向之前，先来看一下cat命令。

    # cat -- 拼接文件

    cat命令 读取 一个或多个文件并将其 复制到标准输出 ：cat filename。

    cat命令常用于显示比较短的文本文件。-n 选项可以显示行号

    当我们想要合并一些文件时，我们可以执行以下命令： cat movie.mpeg.0* > movie.mpeg 通配符会按照顺序扩展，此cat命令的参数排序也不会出错

    # cat命令没有指参数

    如果cat命令**未指定任何参数**，它就会 从__标准输入__中读取 。

    • 执行命令cat，它会一直等待我们从键盘输入，并输入内容输出到屏幕中。
      • 执行ctrl + D可以告诉cat命令已经到达了标准输入的文件末尾End Of File, EOF
    • 由于在缺少参数的情况下，cat命令会 将标准输入__复制__到标准输出 。所以，我们再将 cat命令的输出重定向某个文件，即可实现往文件里写入数据：cat > log.txt。
    • 执行cat log.txt，可以将文件的内容复制到标准输出以查看。
    • cat < log.txt：将标准输入从键盘改为文件log.txt。其结果和cat log.txt是一致的

    # 管道

    通过管道操作符 | ，可以 将一个命令的标准输出传给另一个命令的标准输入 ：command1 | command2。

    之前学过的less就能够使用标准输入！ 对于那些将结果输出到标准输出的命令，我们都可以使用less将其输出结果逐页显示出来： ls -l /usr/bin | less

    # sort --排序列表

    通过sort命令可以实现排序。默认按照字符顺序对比，从左到右，一位一位对比。

    • -n选项：对数字进行排序
    • -r选项：逆序（降序）排序
    • -k2选项：按照第2列进行排序，注意：k要和数字连在一起。 所谓列就是以 空白字符 分割
    • -t 'sep' 选项：设置以 sep 为分割符

    企业面试题案例：对/etc目录下面的文件/目录的大小进行排序（取前10个） 步骤：

    1. ls -l /etc
    2. ls -l /etc | sort -k5
    3. ls -l /etc | sort -rnk5
    4. ls -l /etc | sort -rnk5 | head -- 最终答案
    • 将/bin和/usr/bin目录下的所有文件合并成一个列表，然后排序，并查看结果：ls -l /bin /usr/bin | sort | less。

    # uniq --报告或忽略重复行

    uniq命令通常与sort命令配合使用。uniq命令可以从标准输入或单个文件名参数中获取有序得到数据列表（搭配sort），默认删除所有重复行。

    注意：uniq去重规则，它只处理相邻的！所以其通常会和_**sort**_一起使用

    • **_-c_**选项：输出去重后的内容，及其出现的次数
    ## 大多数情况下， uniq前的sort是为了让重复的排在一起，
## 而uniq之后的sort就是正真意义的排序了

## 查看前10个 去重排序之后的 /bin 目录 以及/usr/bin下的文件列表
ls /bin /usr/bin | sort | uniq | sort | head

    # wc --统计文件中换行符、单词、及字节的数量

    wc全拼为word count，译为单词统计。

    wc ls_output.txt
-> 7902 64566 503634 ls_output.txt
    1
    2

    命令输出的三个数字分别是ls_output.txt中的**行数、单词数、字节数**。

    • 如果没有指定命令行参数，wc命令会从 标准输入中读取。
    • -l选项限制 只输出 文件行数 。
    • -c选项限制只输出 字符数

    企业级应用案例，通过wc命令，统计次数，出现了多少个 ls -l | wc -l：统计当前目录文件/目录的数量

    # grep --输出与模式匹配的行

    grep命令可用于在文件中查找文本模式：**grep **_**pattern filename**_。当grep命令在文件中遇到"模式"的时候，会_输出包含该模式的行_。

    • ls /bin /usr/bin | sort | uniq | grep zip：从程序列表中找出名称中包含单词zip的所有文件。
    • -i 选项使grep命令在搜索的时候 忽略字母大小写
    • -v 使grep命令只输出不匹配指定模式的行
    • -n 显示行号 ，内容在文件中的行号
    • -Ax 除了显示匹配的内容外，还显示其 下面__的x行
    • -Bx 除了显示匹配的内容外，还显示其 上面__的x行
    • -Cx 除了显示匹配的内容外，还显示其 上面和下面__的x行

    # head/tail --输出文件的开头/结尾部分

    • head命令和tail命令默认分别能够输出文件的前10行和后10行
    • -n num选项可以指定输出多少行

    tail -f命令可以实时查看文件，一旦文件添加了新行，立即就会输出出来。

    # tee --读取标准输入并将输出结果写入标准输出和文件

    • tee命令从标准输入读取内容，然后将其 复制到标准输出 和 其他文件 中
    • ls /usr/bin | tee ls.txt | grep zip：在屏幕中打印包含zip的文件名，同时还会创建ls.txt来接收/usr/bin的所有文件名
