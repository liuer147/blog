    • 掌握ls -lhi每一列的含义

    第一列是：inode文件唯一标识，相当于我们的身份证

    第二列是：文件类型和权限信息。第一个字符表示文件类型。后面 9 个字符，分为 3 组，3个字符一组。分别表示 文件所有者、文件所属组、其他用户 的 读、写、执行 权限。

    第三列是：硬链接数量。

    第四列是：文件所有者 的 用户名。

    第五列是：文件所属组的组名。

    第六列是：文件的大小。

    第七-九列是：文件的修改日期。月、日、时间。

    最后一列是：文件名。

    # inode与block

    • inode：存放文件的属性信息(大小、权限、时间、链接数)，包括文件内容block的位置。

    • block：存放数据的空间。文件内容

    # Linux文件类型

    # 扩展名

    • linux：linux下的扩展名是给我们人看的，没有其他意义。
    • windows：通过扩展名区分不同的文件，如果扩展名改变，默认无法识别文件。.txt、.doc、.exe。
    **_Linux_**常见扩展名 作用
    .txt 普通文件
    .cfg、.conf 配置文件
    .sh、.bash 脚本文件

    # 文件类型

    file item...查看文件类型

    Linux 文件类型
    file文件 -
    directory目录 d
    link(softlink、symlink)软链接（符号链接） l -> 大多存在与/dev目录下
    block块设备文件 b -> 磁盘、硬盘、光驱等
    character特殊符号文件 c -> 用于输出（随机数生成）和吸收（垃圾桶）字符
    pipeline管道文件 --了解
    socket套接字文件 -- 了解

    常见特殊符号文件

    • 用于吸收字符：/dev/null：可以将不要的内容，垃圾文件放到此文件里面，就相当于丢弃了。
    • 用于输出字符：/dev/urandom：可以用于生成随机数、密码。
      • tr -cd 'a-z' < /dev/urandom | tail -c10生成a-z之间的10个字符
