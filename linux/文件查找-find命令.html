    # find基本介绍

    选项
    find目录 选项
    -type 查找指定类型的文件：f (file), d (directory)
    -name 指定文件名
    -size 根据大小进行查找：+xx表示大于xx，-xx表示小于xx.
    单位k：表示kb；单位M：表示MB(注意是大写)
    -mtime 根据文件的修改时间进行查找：

    • -mtime +77天前

    • -mtime 7刚好7天时

    • -mtime -77天内 | | -maxdepth | 指定查找的深度，默认递归查找：-maxdepth 1查找1层下的文件。注意：此选项要放在选项的第一个，不然会出现警告 | | -iname | 查找文件的时候不区分大小写（文件名） | | -user或-group | 指定用户或用户组 | | -perm | 指定权限 |

    • 案例1：在/etc/查找文件，文件名叫hostname

    问：为什么find find -type f -name '*.txt' |xargs cp ./命令会出现该错误呢？

    答：因为其类似：cp ./ file... file相当于将中间的复制到最后一个参数，这个时候最后一个参数只是文件，而不是目录。

    • 借助mv、cp的-t选项切换其参数顺序，即可解决

    问：为什么用-exec+{} \;最后打包的文件里面只有最后一个文件呢？ 答：不同于方法1的``，方法2的|xargs这两个是一次性将结果传递给tar命令， -exec是有一个就传一个，有一个结果就执行一次-exec后面的命令，所以每次压缩的结果都是覆盖了前一次的

    • 将{} \;替换为+即可解决
