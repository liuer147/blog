    Linux用户分类

    # Linux用户分类

    • uid用户身份证号码
    • gid用户的户口本号码
    用户分类 含义 uid
    root用户 最高权限用户 0（只要uid为0，那么他就是root用户）
    普通用户 只对自己家目录拥有权限的用户 1000开始（CentOS7.8）（CentOS 6是500）
    虚拟用户（傀儡用户） 这种用户存在适用于让一些服务正常运行，一些服务运行需要指定用户 0-999（非唯一判断标准）

    # 用户相关文件

    • /etc/passwd：存放用户信息（非密码信息）
    • /etc/group：存放用户组的信息
    • /etc/shadow：存放用户的密码信息
    • /etc/gshadow：存放用户组的密码信息
    • /etc/profile：环境变量（PS1），别名配置文件（alias）
    • /etc/bashrc：别名配置文件
    • ~/.bashrc
    • ~/.bash_profile

    查看系统有多少个用户，可以查看/etc/passwd有多少记录

    # /etc/passwd每一列的含义

    • 虚拟用户的家目录路径各种情况都有，没有规律可言。其家目录不一定存在
    • 判断是否是虚拟用户的重要依据是命令解释器
      • /bin/bash(CentOS) or /bin/dash(Ubuntu)：root用户或普通用户
      • /sbin/nologin ：虚拟用户
    • 将密码标记列的x删掉的话，对应用户就没有密码了

    # 用户相关目录

    # 故障案例

    • 现象： 输入命令的时候，命令行提示符[用户名@liuer ~]$变成-bash-4.1$或-bash-4.2$样子。
    • 原因： 用户不小心删除了当前用户家目录下面的环境变量相关的文件.bash*。
    • 解决： 从/etc/skel/.bash*复制到当前用户家目录，重新登录即可。

    # 用户管理命令

    # 增加用户

    • useradd添加用户：基本用法useradd xxx
    • adduser添加用户
    选项
    -s 添加用户的时候指定用户的命令解释器，如果不指定则默认是/bin/bash
    -M 添加用户的时候不创建家目录。(一般与-s一起使用-Ms)
    -u 添加用户的时候指定uid
    -r 添加系统用户（不创建目录，_uid 1-999_，命令解释器_bash_）
    -G 附加组（额外组）：添加用户的时候，让用户属于多个组
    -g 主要组（原生组）：替换默认创建的组
    [root@liuer ~]$ ll /usr/sbin/useradd /usr/sbin/adduser
lrwxrwxrwx 1 root root      7 Aug 19  2021 /usr/sbin/adduser -> useradd
-rwxr-xr-x 1 root root 147344 Aug 19  2021 /usr/sbin/useradd

    # 切换用户

    • su切换用户switch user -> 格式：su - xxx
    • ctrl + d退出当前用户，可以一直操作知道初始登录者
    su选项
    -，-l，-login 切换用户的时候，更新下用户的环境变量

    # 修改密码

    • passwd [user]：修改指定用户或当前用户的密码
    • 非交互式设置密码(只有root用户才可以执行此操作)：echo '密码' | passwd --stdin 用户。

    # 删除用户

    • userdel xxx ：删除用户的时候只删除用户的信息，不会删除用户的家目录，邮件文件...
    • userdel -r xxx：彻底删除用户，包括用户的家目录，邮件文件。（不推荐）

    不推荐删除用户！！ 如果非要删除，可以在**/etc/passwd**中注释掉用户

    # 修改用户信息（了解）

    • usermod：修改已存在的用户的信息，useradd添加用户时候设置的信息，usermod都可以改。
    • usermod：它的选项和useradd类似。
    ## 修改用户组
[root@liuer ~]$ id test
uid=1001(test) gid=1001(test) groups=1001(test)
[root@liuer ~]$ usermod -G 'root' test
[root@liuer ~]$ id test
uid=1001(test) gid=1001(test) groups=1001(test),0(root)

    # 查看用户信息

    • 查看用户基本信息
      • whoami：显示当前用户的名字
      • id：查看当前或指定用户的信息（uid、gid、用户组信息）
    • 查看用户登录情况
      • w

    • last：查看用户的登录情况。
    • lastlog：查看所有用户的最近一次登录情况。

    # 用户组命令(了解)

    • groupadd：添加用户组
    #案例 企业案例				添加用户的时候保证用户和用户组uid，gid一致

如果直接添加的时候uid，gid一致，那么需要先添加用户组，然后创建用户
groupadd -g 8888 mysqlv2										# -g指定gid
useradd -u 888 -g mysqlv2 mysqlv2 					# -g指定用户组
