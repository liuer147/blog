    像开始所说的通配符，比如*这样简单的字符在shell中意味着诸多含义。这一处理过程被称作扩展。经过扩展，我们输入的内容在被shell执行之前会被展开成其他内容。

    • echo xxx ：将文本参数 xxx 输出到标准输出

    问：echo *命令会将当前目录下的文件名输出到标准输出，为什么没有输出*？ **答：**在执行echo命令之前，由shell将*扩展为其他内容（在这里是当前工作目录中的文件名）。按enter键时，shell会在命令 执行之前 自动扩展命令行中的所有限定符

    # 路径名扩展

    通配符的工作机制被称为路径名扩展。

    ## 将当前目录下以 "D"开头 的文件名输出到标准输出流中
echo D*

## 将当前目录下以 "s"结尾 的文件名输出到标准输出流中
echo *s

## 将当前目录下以 大写字母开头 的文件名输出到标准输出流中
echo [[:upper:]]*

    # 浪纹线扩展

    浪纹线~用于单词之前时，会被扩展为同名用户的主目录。如果未指定单词，则扩展为当前用户的主目录。

    echo ~ 				## -> /root
echo ~foo			## -> 存在对应用户的话则是 /foo。不存在则直接输出 ~foo

    # 算术扩展

    算术扩展采用这种形式：$((expression))。

    • 算术表达式仅支持整数（不能是小数）。
    • 算术表达式中的空格没有特别意义。
    • 可使用圆括号分组多个表达式

    支持的算术操作符有：

    操作符 含义
    + 加法
    - 减法
    * 乘法
    / 除法 -- 结果会丢弃小数位
    % 求模，就是求"余数"
    ** 求幂
    echo $((2 + 2))			##-> 4
echo $(((5**2) * 3))		##-> 75
echo $((5 / 2))			##-> 2    它会丢弃小数位

    # 花括号扩展

    {}花括号扩展可以按照括号中的模式创建多个文本字符串。

    echo var-{a,c,e}		##-> var-a var-b var-c
echo var-{a, c, e}		##-> var-{a, c, e}     花括号中不能包含未经引用的空白字符

echo {01..15}			##-> 01 02 03 04 05 06 07 08 09 10 11 12 13 14 15
echo {001..15}			##-> 001 002 003 004 005 006 007 008 009 010 011 012 013 014 015

echo {Z..A}		##-> Z Y X ... C B A 逆序输出字母表

echo a{{1,2},{3,4}}b 		##-> a1b a2b a3b a4b

    # 命令替换

    命令替换允许我们使用命令的输出结果作为扩展结果：

    echo $(ls)		##-> 将ls命令的结果输出到标准输出 => 等价于 echo `ls`
file $(ls)		##-> 将ls的结果作为file 的参数 => 等价于 file `ls`

    # 双引号

    如果将文本放入双引号，那么除了$(美元符号)、\(反斜线)、``(反引号)`之外的所有字符将失去其特殊含义。

    ls -l two words.txt			##-> 此时 two words.txt为两个参数
ls -l "two words.txt"			##-> 正常

正常情况下单词分割会查找空格符、制表符、换行符。将其视为单词之间的分隔符。双引号禁用了单词分割
