    # new操作符

    执行过程：

    1. 创建一个原型[[prototype]]指向构造函数prototype的对象
    2. 将构造函数绑定这个新对象，并执行
    3. 如果执行后的结果是一个非空对象，则返回这个非空对象；否则，返回开始创建的新对象
    function myNew(func, ...args) {
  const obj = Object.create(func.prototype)
  const result = func.call(obj, ...args)
  if(typeof result === 'object' && result != null) {
    return result
  }
  return obj
}

    # 对象属性

    如果一个属性既存在于对象中，又存在与此对象的原型中[[prototype]]。那么，对象本身的属性则会屏蔽原型上的同名属性。 当我们访问对象的该属性时，当我们修改对象的该属性时，都是操作的对象本身的属性。

    即使将对象属性设为null、undefined，再次访问时，仍然 **_不是 _**其原型上的属性值

    注意：

    • 我们不能通过对象来修改其原型上的属性值。因为我们在给对象赋值时，会在其本身添加对应属性；
      • 看似修改成功，是因为对象本身的属性覆盖了其原型上的属性

    # 判断某个属性只存在于原型上，不存在与对象上

    注意：此方法只能判断【只】存在于原型上，而不能判断【是否】存在于原型上。

    • 当对象存在某个属性，且原型也存在某个属性时，此方法仍然返回false
    • 如果想判断是否在原型上存在某个属性，可以使用Object.getPrototypeOf(obj).hasOwnPropertyOf(name)
    function isPropertyInPrototype(obj, name) {
	return name in obj && !obj.hasOwnProperty(name)
}

    # 对象属性获取

    • for(const i in obj)：此处的i为对象obj【及_对象原型__**[[prototype]]**_】中**可枚举的属性名**，不包括**_Symbol_**属性。
    • Object.keys、Object.values、Object.entries：获取**对象本身的、可枚举的、不包括**_Symbol_**属性**的对应的值
    • Object.getOwnPropertyNames()：获取**对象本身的、不包括**_Symbol_**属性**的所有属性【不可枚举的也存在】
    • Object.getOwnPropertySymbols()：获取对象的符号属性

    # 对象属性的枚举顺序

    • for...in、Object.keys的枚举顺序是**不确定的**。
    • Object.getOwnPropertyNames、Object.getOwnPropertySymbols、**_Object.assign_**的枚举顺序是确定的。
      • 先以升序枚举数值键，然后以插入顺序枚举字符串键和符号键

    # 作用域链增强

    • 虽然执行上下文主要有**全局上下文和函数上下文**两种（**_eval()_**内部存在第三种上下文）。
    • 但是某些语句会导致在作用域链前端临时添加一个上下文，这个上下文在代码执行后会被删除。
      • **_try...catch_**语句的**_catch_**块
      • **_with_**语句

    **注意：**这两个上下文并不会影响 this 的指向。

    # 变量声明

    • 使用var声明变量时，变量会被自动添加到_最接近的上下文对象_。
    • 如果_变量未经声明就被初始化了_，那么它就会自动被添加到**全局上下文**。
    • let与var的区别：
      • let声明的变量是块级作用域的，var声明的变量是函数作用域。
      • let同名变量在同一作用域内不能声明两次，var则会静默忽略重复的声明。
      • 严格来说，let声明也会在运行时提升，但由于【暂时性死区】，实际上不能在声明之前使用该变量。
        • var变量提升，其被初始化为undefined，let/const变量提升，不会初始化为任何值，由于【暂时性死区】，需要等到赋值初始值之后才能访问。

    # 对象解构

    • 对象解构在内部会【将_被解构目标_转化为_对象_】。也就是说：
      • 目标是原始值的话，会转换成对象。
      • 而null和undefined不能被解构【如下图】，否则抛出错误。
      • 如果对象本身不存在该属性，则会去原型上找。

    解构操作也是不能回滚的。如果解构过程中抛出错误，则整个解构也会完成一部分

    # nodeValue、textContent、innerText

    • nodeValue (opens new window) 指的是结点的值。对于Document、Element来说就是固定的null，即使手动对其赋值，也是会被忽略。对于文本结点、注释结点来说，其就是对应的文本内容，赋值即修改。
    • textContent (opens new window) 指的是结点及其后代结点的文本内容。其会将元素及子元素的所有文本内容组合在一起返回。
    • innerText (opens new window) 指的是结点及其后代结点 _渲染 _的文本内容。也就是说，对于display:none;或者visibility: hidden;样式的文本（只要没有渲染到页面上）就不会返回。
      • 由于其受CSS样式影响，所以我们访问一个元素的innerText时，会触发浏览器的reflow。

    注意：一旦设置元素的textContent、innerText这两个属性，其原本的所有子内容都会删除，从而由新的对应的值替换。
