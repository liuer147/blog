    参考：错误处理，“try...catch” (opens new window)

    • **catch**** 应该只处理它知道的 **error**，并“抛出”所有其他 **error**。**

    • **finally**语句不管如何都会执行，如果**try**或**catch**中显式**return**。

      • 那么到达return语句时，也会先执行finally块中的内容。
      • 如果finally块中也显式return，则最终return的是finally块中的【覆盖了try、catch】

    • **try...catch**无法捕获异步错误

    • Error对象包含下列属性：

      • message人类可读的 error 信息。
      • name 具有 error 名称的字符串（Error 构造器的名称）。
      • stack（没有标准，但得到了很好的支持）—— Error 发生时的调用栈。

    • 全局捕获错误

      • window.onerror = function(message, url, line, col, error)
