    # Proxy 只是相当于针对 目标对象 包裹了一层代理。

    也就是说，如果我们直接取得其内部的属性引用。那么通过这个引用来执行某个操作，则会出现无法跟踪的现象。

    const originObj = {
  name: 'originObj-name',
  ref: {
    age: 22
  }
}

const proxyObj = new Proxy(originObj, {
  get(target, key ,receiver) {
    console.log('代理: get')
    return Reflect.get(target, key ,receiver)
  },
})
const childRef = proxyObj.ref // 访问一次，打印

console.log('---------------------')

console.log('originObj.name', proxyObj.ref.age) // 访问一次，打印


console.log('childRef.age', childRef.age)

    age: 22

const proxyObj = new Proxy(originObj, {

proxyObj.name = 'new name' // 打印：代理: set =>  new name
proxyObj.ref.age = '999' // 没有触发 set

    # 为什么在 Proxy 捕获器中通常是返回对应 Reflect 相同方法。而不是直接返回 target[key] 呢？

    首先，需要使用 Reflect 来执行操作的是在 get(target, key, receiver)、set(target, key, value, receiver)捕获器中。 其中 get捕获器参数receiver是指代理的对象，或者 继承自代理对象的对象；set捕获器参数receiver是指原始调用对象。

    而 Reflect 的方法签名 和 Proxy捕获器对象的方法签名一一对应。根据 ecma262 规范，调用 Reflect.get、Reflect.set时，如果对应 target[key] 是访问器属性时，则会将访问器的this绑定到receiver上执行，从而可以保证访问器属性中this的正确指向。 而有时，Proxy 的捕获器属性需要返回值 true/false 来表示是否成功。而 Reflect 对应方法执行后也会返回相应的布尔值。

    # Proxy 的局限

    一些 ECMAScript 内置类型方法 的执行依赖于其内部的槽位。而 Proxy 代理的对象上不存在这个内部槽位，且也无法通过普通的get()、set()访问的。所以 Proxy 无法拦截。

    Map、Set、Date、Promise等都是用到了 "内部槽位"。

    const target = new Date()

const proxy = new Proxy(target, {})

console.log(proxy instanceof Date) // true

proxy.getDate() // TypeError

    # 可取消的 Proxy

    let {proxy, revoke} = Proxy.revocable(target, handler)
    1
    let object = {
  data: "data"
};

let {proxy, revoke} = Proxy.revocable(object, {});

// proxy 正常工作
alert(proxy.data); // Valuable data

// 之后某处调用
revoke();

// proxy 不再工作（已吊销）
alert(proxy.data); // Error
