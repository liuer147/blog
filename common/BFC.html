    参考：https://developer.mozilla.org/zh-CN/docs/Web/Guide/CSS/Block_formatting_context (opens new window)

    块级格式化上下文（Block Formatting Context，BFC）是 Web 页面中用于布局和定位块级盒子的**一种 CSS 渲染模式**。**通过创建 BFC，可以让环境内部的布局不会影响到外部的元素，同时内部的元素也不会受到外部的影响**。

    # 创建 BFC

    创建块级格式化上下有很多，具体可以查阅 MDN 网址。这里简单介绍几种：

    1. 文档根元素<html>
    2. 脱离文档流的元素。float、position为absolute、fixed
    3. 行内块元素。display:inline-block;
    4. overflow不为visible、clip的块级元素。
    5. 模仿根元素display: flow-root;。可以创建无副作用的块级格式化上下文。

    # BFC 的作用

    • 阻止外边距重叠。
    • 包含内部浮动。（块盒子不再因为内部元素浮动脱离文档流，而导致的高度塌陷）（也可以通过浮动元素的伪元素，或者相邻的其他元素设置[clear](https://developer.mozilla.org/zh-CN/docs/Web/CSS/clear)来清除浮动，并自动添加能包容浮动元素的外边距（向下移动）。其他元素不影响，仍然受浮动影响，不过仍然会受这个清除浮动的元素影响）
    • 排除外部浮动。（外部元素的浮动，造成的脱离文档流，对拥有 BFC 的块盒子不会造成影响，仍然会当其存在。尽管其仍然不在标准文档流中）
    <style>
  #container {
    // 如果不为父元素创建 BFC, 则默认不会包裹内部的浮动元素，从而导致高度塌陷
    display: flow-root; 
    border: 10px solid greenyellow;
  }
  .float {
    float: left;
    width: 100px;
    height: 100px;
    border: 1px solid red;
  }

  .common {
    border: 1px solid;
    background-color: aqua;
    width: 200px;
    // 如果不为common元素创建 BFC，则其会因为浮动元素脱离文档流（当其不存在），而自动向包含块的边沿对齐。
    // 如果创建了 BFC，则不会因为此原因而自动对齐
    display: flow-root; 
  }
</style>

<div id="container">
  <div class="float">浮动元素</div>
  <div class="common">普通元素普通元素普通元素普通元素普通元素普通
  </div>
</div>
