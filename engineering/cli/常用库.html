    # commander

    为了简化命令行的解析操作，更多的去关注逻辑执行。我们将引入 Commander (opens new window) 第三方库来处理命令行参数，执行以下命令进行安装：

    npm install commander
    1

    commander提供了一个全局对象program，可以快速解析命令行。也提供了一个Command构造函数。当程序较为复杂时，可以通过Command构造函数来创建多个program。

    const { program, Command } = require('commander')

// const program1 = new Command()

program.parse(process.argv) // 解析命令行参数

    当我们使用program.parse解析命令行参数时，它会默认帮我们注册脚手架的**--help**指令。输入并执行hx --help，打印如下：

    我们会发现，--help指令在Options下面。为什么会这样呢？后面再说 值得一提的是：在Commander中有一个潜规则，所有的**"选项"应该是"-缩写，--全拼"的形式，对于"命令"**就是不以"-"开头。当然，这并不是绝对，只是推荐。

    什么是选项？什么是命令？它们有什么区别？ 就目前为止，我们只需要明白它们都是指令，至于有什么区别，后面再说

    现在，我们尝试用commander来实现开始的"-V，--version"，"ls"，"ls -a"。 代码如下，

    • program.command用于注册命令，当用户输入的内容和这里注册的ls相匹配时，就会执行后续的.action(handler)的handler回调函数。
    • option用于注册选项，其第二个参数是选项的描述。
    • description是前面命令的描述。
    1. 当存在command和option时，option可以认为是"command下的option"，此时的option只有在command存在的情况下才生效。同样的当执行hx ls --help获取指令的帮助(commander默认帮我们注册的)时，可以在options中看到。
    2. 在action之前注册的command的 参数 会依次传入.action(handler)的handler回调。 而注册的所有option都会放到handler的command参数后一个参数对象中，如果说选项的形式为"-缩写，--全拼"则对应在参数对象中的属性名为"全拼"，如果是"-xx"那对应的属性名就是xx。
    const { program } = require('commander')

// .version是专门设置版本号的，
// 其默认选项为-V和--version，设置了版本后，命令行会输出当前的版本号。
program.version(require("./package.json").version)

program
    .command('ls')
    .option('-a', '获取当前目录的所有文件列表')
    .description('获取当前目录的普通文件列表')
    .action((options) => {
        const { a } = options
        const files = fs.readdirSync(process.cwd())
        const fileStr = a ? files.join(' ') : files.filter(item => !item.startsWith('.')).join(' ')
        console.log(fileStr)
    })
/*
// 正如上面所说的，如果command 里面的指令存在传入参数，则这个参数就对应action回调的前面几个参数。
// 而 option 则是指令对应的参数的后面的一个参数
program
    .command('create  <name>')
    .description('拉取远程git仓库')
    .action((name) => {
        // 暂时写死RuoYi-Vue地址

*/

    # 编写create命令

    先提需求：我们编写的这个create命令可以拉取git仓库的代码， 首先，我们先使用program.command注册create命令，并且使用description给它加上描述，我们先在action中打印"create"。代码如下

    const { program } = require('commander')

// .version是专门设置版本号的，
// 其默认选项为-V和--version，设置了版本后，命令行会输出当前的版本号。
program.version(require("./package.json").version)

program
    .command('ls')
    .option('-a', '获取当前目录的所有文件列表')
    .description('获取当前目录的普通文件列表')
    .action((options) => {
        const { a } = options
        const files = fs.readdirSync(process.cwd())
        const fileStr = a ? files.join(' ') : files.filter(item => !item.startsWith('.')).join(' ')
        console.log(fileStr)
    })

/** create命令 */
program
    .command('create')
    .description('拉取远程git仓库')
    .action(() => {
        console.log('create')
    })

program.parse(process.argv)

    类似vue-cli的vue create xxx那样，我们需要用户输入拉取git代码之后存放在本地的文件夹的名字。那么，怎么让我们注册的这个create命令可以捕获其后面的内容呢？ 修改create指令代码如下：

    /** create命令 */
program
    .command('create <name>') // 此处传递给command变为create <name>
    .description('拉取远程git仓库')
    .action((name) => { // command捕获后面的内容传递给action(handler)的handler作为参数
        console.log('create ', name)
    })
    1

    我们由开始的.command('create')变为了.command('create <name>')。它会捕获用户输入的内容中create后面的那个指令，作为这里的"name"并传递到action(handler)的handler。

    值得一提的是：

    • 对于create后面<>这种形式的话，表示是必选的，也就是说当用户在输入create时，其后面必须跟一个内容：create xxx。如果不跟，则无法执行create命令。
    • 如果需要是可选的。也就是输入create命令后面可写可不写的话，是要使用[]形式。也就是这样：.command('create [name]')

    .option也是如此：.option('-a, -all <xxx>')。不过option最多只解析一个参数。多的参数无效

    .option和.command的区别就是：对于捕获的内容（以后统称为参数，command捕获的就是command参数，option捕获的就是option参数），command参数会依次传递给action(handler)的handler。而option则都放在command传递给handler的参数后面一个的参数对象中。示例如下：

    program
    .command('aa <arg1> [arg2]')
    .option('-B, --bb <op1> [op2]')
    .action((arg1, arg2, arg3) => {
        console.log(arg1, arg2, arg3)
    })

    接下来，我们来处理模板地址。开始，我们是写死的若依的地址，这显然太局限了。我们希望可以让用户自己来选择git地址，为此，我们将引入inquirer (opens new window)第三方库。它主要解决命令行与用户的交互问题。 执行npm i inquirer@^8.0.0可以下载。 inquirer.prompt用户创建与用户交互的问题。它接收一个对象数组，数组项对象视为Question类实例。返回一个Promise，它有以下属性(只写出一部分，等多请查阅官网)：

    • type：问题类型，默认是input。支持input(文本输入)、number(数字输入)、confirm(确认)、list(单选列表)、checkbox(多选列表)、password(密码输入)...更多请查阅官网
    • name：用户输入的值对应的变量名。对应.prompt(...).then(answer)中answer的属性
    • message：问题的描述
    • default：问题答案的默认值
    • choices：选项数组。数组项可以是Number，String，{ name: '', value: '' }形式
    • 的对象

    实例如下：

    const inquirer = require('inquirer')

inquirer.prompt([
  {
    type: 'input',
    name: 'test1',
    message: '请输入：'
  },
  {
    type: 'list',
    name: 'test2',
    message: '请选择',
    choices: ['answer1', 'answer2']
  }
]).then(answer => {
  const { test1, test2 } = answer
  console.log(test1, test2)
})

    const { program } = require('commander')
const download = require('download-git-repo')
const ora = require('ora')
const chalk = require('chalk')
const inquirer = require('inquirer')

// .version是专门设置版本号的，
// 其默认选项为-V和--version，设置了版本后，命令行会输出当前的版本号。
program.version(require("./package.json").version)

program
    .command('ls')
    .option('-a', '获取当前目录的所有文件列表')
    .description('获取当前目录的普通文件列表')
    .action((options) => {
        const { a } = options
        const files = fs.readdirSync(process.cwd())
        const fileStr = a ? files.join(' ') : files.filter(item => !item.startsWith('.')).join(' ')
        console.log(fileStr)
    })

/** create命令 */
program
    .command('create  <name>')
    .description('拉取远程git仓库')
    .action((name) => {
        const tempArr = [
            'direct:https://gitee.com/y_project/RuoYi-Vue.git', // 诺依
            'direct:https://gitee.com/element-plus/element-plus.git', // elementPlus

                  spinner.succeed('下载成功') // 成功
                }
            })
        })
    })

program.parse(process.argv)
